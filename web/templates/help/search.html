Diagram Search

KnotEditor searches diagrams in a network database located on this site. The server does not store the diagrams; it generates the layout dynamically, using Spherogram and circle packing algorithm implementation from David Eppstein PADS library.

Please note that the server does not distinguish a knot from its mirror, so the response might be the mirror of the knot you expect.

Search Pattern Syntax

For network diagram search, the server supports several pattern formats. Some of them are listed below.

  • DT code

    A space-separated list of even numbers, e.g. 6 20 22 -14 2 4 -18 -8 -12 -16 10, or a string of Latin letters like cjkGabIDFHe. If you don't know what the DT code is, please read the Wikipedia article.

  • Torus knots

    To load a diagram of a torus knot, use T(p,q) notation, where p and q are integers. E.g., T(2,3) is the trefoil.

  • Prime knots up to 10 crossings (Rolfsen table)

    N_k, where N and k are integers, denotes k-th knot with N crossings. E.g., 4_1 is the figure-eight.

  • Prime knots up to 16 crossings (Hoste-Thistlethwaite-Weeks table)

    The knots are listed in “The first 1,701,936 knots” article. We use the notation taken from the Knot Atlas. A knot is denoted as K<num>[an]<idx>. Here <num> is the number of crossings, an integer between 3 and 16; a means the knot is alternating, n is for non-alternating; and finally <idx> is the index of the knot in the list ordered by minimal DT code.

    E.g., K3a1 is the trefoil, and K8n3 is the third non-alternating prime knot with eight crossings.

  • Regular expressions

    Special character ? means “any character”, * is for “any sequences of characters”. E.g., query 9_? matches nine knots, from 9_1 to 9_9. Query 9_* matches 49 knots.

    Note that regular expressions work with knot tables only. DT codes, as well as Torus knot identifiers, cannot be used with special characters.
